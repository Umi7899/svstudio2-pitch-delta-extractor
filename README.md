# 提取渲染音高到音高偏差

这是一个 Synthesizer V Studio 2 的脚本。  
用途是把当前编辑组的自动渲染音高提取到“音高偏差”参数面板。

## 功能说明

- 支持“选中音符优先”（未选中时自动回退为当前组全部音符）
- 写入前清空目标区间已有音高偏差，避免叠加污染
- 对断开的音符之间的无音符区间强制写入 `0`，防止前后曲线自动连线
- 给每个音符补起止锚点，避免前一个音符影响后一个音符的初始偏差
- 当某段拿不到可用渲染音高（如歌词为空、该段无渲染结果）时自动写 `0`，不中断全流程

## 使用方式

1. 打开目标工程并进入要处理的编辑组。
2. 运行脚本“提取渲染音高到音高偏差”。
3. 在弹窗中设置范围、采样档位和边界策略。
4. 点击确定后，脚本会把结果写入“音高偏差”。

## 参数详细说明

### 作用范围

- 选中音符优先（无选中则全部）：
  有选中音符时只处理选中区域；没有选中音符时处理当前组全部音符。
- 当前组全部音符：
  无论有没有选中，都按当前组全部音符处理。
- 仅选中音符：
  必须先选中音符，未选中会直接提示并退出。

### 采样档位

- 平衡（5ms，低强度简化）：
  默认档位，精度和速度平衡，适合绝大多数情况。
- 高精度（2ms，最小简化）：
  点更密集，细节更完整，但处理更慢。
- 高性能（10ms，中等简化）：
  处理更快，细节会更少，适合快速批量处理。

### 边界策略

- 自动外扩 1/16 拍：
  在目标边界前后各扩一点，尽量保留转折处的过渡信息。
- 严格按选择边界：
  只处理你选中的精确时间范围，不做外扩。
- 扩展到连通乐句：
  如果选中某个乐句中的一部分，会扩展到整段连通乐句一起处理。

### 纯 AI 基线

- 此选项固定开启。
- 行为是先清空目标区间旧“音高偏差”，再写入本次提取结果。
- 目的：确保提取结果基于当前渲染，不受历史手工曲线影响。

## 结果提示说明

脚本完成后会弹出统计信息：

- 区间数：本次实际处理的时间区间数量
- 采样帧数：参与计算的采样帧总数
- 可见控制点：最终写入后在面板可见的控制点数量
- 限幅帧数：超出安全范围被裁剪的帧数量（越接近 0 越好）
- 耗时：本次处理时间

## 注意事项

- 脚本只处理“音高偏差”参数，不改动其他编辑参数。
- 如果同一个音符组在工程中被多处引用，确认执行后会同时影响这些引用位置。
