# 提取渲染音高

这是一个 Synthesizer V Studio 2 的脚本。  
它可以把当前编辑组的自动渲染音高提取出来，并写回到可编辑的音高参数中。

## 功能说明

- 支持“选中音符优先”，未选中时自动回退为当前组全部音符
- 支持两种输出模式：
  `音高偏差` 和 `音高控制曲线`
- 写入前可清空目标区间内上一次提取结果，避免叠加污染
- 对断开的音符之间写入保护点，尽量避免曲线跨休止区自动连线
- 连续批量处理时会自动管理异步轮询和宿主对象刷新，避免 `invalid state`
- 当某段拿不到可用渲染音高时，会自动跳过或回退，不中断全流程

## 输出模式

### 音高偏差（相对 cents）

- 把提取结果写入 `pitchDelta`
- 适合继续沿用“音高偏差”面板的工作流
- 由于 `pitchDelta` 是按当前音符的相对偏差解释，连续大跳音程的衔接处不一定能完整保留绝对音高走势

### 音高控制曲线（绝对音高覆盖）

- 把提取结果写入 `PitchControlCurve`
- 脚本会自动把涉及到的音符切换为手动音高模式
- 更适合把一个声库的渲染音高迁移到另一个声库上
- 如果你的目标是“提取当前声库音高，再切换到别的声库继续使用”，推荐优先使用这个模式

## 使用方式

1. 打开目标工程并进入要处理的编辑组。
2. 运行脚本“提取渲染音高到音高偏差”。
3. 在弹窗中设置作用范围、输出模式、采样档位和边界策略。
4. 点击确定后，脚本会把结果写入所选输出模式对应的目标参数。

## 参数说明

### 作用范围

- 选中音符优先（无选中则全部）
  有选中音符时只处理选中区域；没有选中音符时处理当前组全部音符。
- 当前组全部音符
  无论有没有选中，都按当前组全部音符处理。
- 仅选中音符
  必须先选中音符，未选中会直接提示并退出。

### 采样档位

- 平衡（5ms，低强度简化）
  默认档位，精度和速度平衡，适合大多数情况。
- 高精度（2ms，最小简化）
  点更密集，细节更完整，但处理更慢。
- 高性能（10ms，中等简化）
  处理更快，细节更少，适合快速批量处理。

### 边界策略

- 自动外扩 1/16 拍
  在目标边界前后各扩一点，尽量保留转折处的过渡信息。
- 严格按选择边界
  只处理选中的精确时间范围，不做外扩。
- 扩展到连通乐句
  如果只选中了乐句的一部分，会扩展到整段连通乐句一起处理。

### 写入前先替换目标区间内之前的提取结果

- 勾选时，会先清理本脚本此前写入到目标区间内的结果，再写入本次提取内容
- 关闭时，不会主动清空已有提取结果

## 结果提示

脚本完成后会弹出统计信息。

- 音高偏差模式：
  区间数、采样帧数、可见控制点、限幅帧数、耗时
- 音高控制曲线模式：
  区间数、采样帧数、新增曲线数、曲线点数、切换为手动音高的音符数、耗时

## 注意事项

- 如果同一个音符组在工程中被多处引用，执行后会同时影响这些引用位置
- 音高控制曲线模式会改动相关音符的音高手动/自动状态
- 如果你的目标是跨声库复用音高，优先使用“音高控制曲线”模式
